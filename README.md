# SQL DDL Practice Project

## Overview

This project demonstrates fundamental SQL Data Definition Language (DDL) operations through a simple retail inventory management case study.

The objective is to practice creating and modifying database structures while simulating common business requirements.

---

## Scenario

A retail store wants to maintain digital records for:

* Customers
* Products
* Inventory

As business requirements evolve, the database schema must be updated accordingly.

---

## Concepts Covered

### Database Operations

* CREATE DATABASE
* USE

### Table Operations

* CREATE TABLE
* ALTER TABLE
* DESCRIBE
* RENAME TABLE
* TRUNCATE TABLE
* DROP TABLE

### Schema Modifications

* Add new columns
* Modify column data types
* Rename columns
* Rename tables

---

## Project Tasks

1. Create a database for the store.
2. Create customer and product tables.
3. Insert sample records.
4. Add new attributes to existing tables.
5. Modify column definitions.
6. Inspect table structures.
7. Rename tables.
8. Remove all records while preserving table structure.
9. Delete tables when no longer needed.

---

## Learning Outcomes

By completing this project, you will learn how to:

* Design basic relational database schemas.
* Use SQL DDL commands effectively.
* Modify database structures based on changing requirements.
* Understand the difference between TRUNCATE and DROP operations.
* Apply SQL concepts to a realistic business scenario.
